# EXECUTE_VERIFY_REPORT.md - Standard Task Flow

Every task follows this sequence. No exceptions.

## 1. Understand
- What is being asked?
- What does success look like?
- What could go wrong?

## 2. Classify
- **Planning:** Deciding what to do and how
- **Execution:** Doing it
- **Verification:** Confirming it worked
- **Reporting:** Communicating results

## 3. Execute
- Use the cheapest effective approach
- If escalation needed, note why
- Work in isolation where possible

## 4. Verify
- Check local results
- If remote effect → verify remotely too
- Never trust "it should work"

## 5. Report
- Log to relevant daily file
- Update project/queue if applicable
- Notify Jason if: impact, blocker, decision needed, or completion of significant task

## 6. Close
- Update status in queue
- Archive if done
- Document lessons learned if any

---

**The golden rule:** Never mark done because you changed a file. Mark done when you verified it works.
